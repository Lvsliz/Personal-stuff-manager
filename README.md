# 拾物清单

一个适合单人生活场景的本地物品管理工具。可直接本地运行，也可作为 PWA 安装到桌面，数据保存在浏览器本地。

## 已实现功能

- 物品录入：名称、类目、品牌、存放位置、单位、数量、低库存阈值、启用日期、过期日期、备注
- 采购管理：购买日期、购买件数、总价、自动计算均价
- 库存动作：消耗、补货、启用、移动位置
- 智能提醒：低库存、30 天内临期、已过期
- 数据总览：库存总量、已开封数量、库存金额、最近采购与变动日志
- 数据能力：JSON 导入 / 导出，便于本地备份
- 安装能力：支持离线缓存和桌面安装
- 兼容能力：导出文件包含 `schemaVersion`、`appVersion`、`exportedAt`，为后续升级迁移预留兼容机制

## 使用方式

1. 用本地静态服务器打开 [index.html](C:\Users\Lvhao\Documents\New project\index.html)。
2. 首次打开后，浏览器若支持 PWA，会显示“安装到桌面”按钮。
3. 先添加几个常用物品。
4. 每次消耗、补货或挪位置时，在“更新库存”里记录一次即可。

## 启动方式

推荐使用项目自带启动脚本，否则部分浏览器不会启用 Service Worker 和安装能力。

最简单方式：

```text
双击 start.bat
```

或在 PowerShell 中执行：

```powershell
.\start.ps1
```

默认地址：

```text
http://localhost:4173
```

如果你的电脑装了 Python，可以在项目目录执行：

```powershell
python -m http.server 4173
```

然后访问：

```text
http://localhost:4173
```

## 数据兼容说明

- 导出文件不是裸数组，而是带元数据的版本化 JSON
- 应用启动和导入时都会先执行迁移函数 `migrateDataShape`
- 以后如果我继续升级字段结构，可以直接在迁移函数里增加旧版本到新版本的转换逻辑
- 建议升级前先导出一次备份，再导入到新版本
